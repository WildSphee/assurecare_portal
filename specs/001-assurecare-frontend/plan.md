# Implementation Plan: AssureCare Portal Frontend

## Summary

Build a modern React + TypeScript single-page application (SPA) with two role-based views вЂ” **Caregiver (Ana)** and **Doctor (Dr. Chan)** вЂ” surfacing structured daily health insights derived from a home voice chatbot. All data is mocked in-memory. The app uses Vite for the build toolchain, shadcn/ui + Tailwind CSS for the component/styling layer, Recharts for trend charts, react-leaflet for the map view, and Zustand for global mock-data state. No backend is required.

---

## Technical Context

**Language/Version**: TypeScript 5.x with React 18
**Primary Dependencies**: Vite 5, React Router v6, shadcn/ui, Tailwind CSS 3, Recharts, react-leaflet + Leaflet, Zustand, date-fns, Lucide React
**Storage**: In-memory mock data only (static TypeScript modules under `src/data/mock/`)
**Testing**: Vitest + React Testing Library (unit/component tests)
**Target Platform**: Modern desktop browsers (Chrome, Firefox, Edge, Safari); minimum 1024px viewport
**Project Type**: Frontend SPA (web-app)
**Performance Goals**: Dashboard load < 3s broadband; Doctor panel (10 patients) < 3s
**Constraints**: No real API calls; no backend; all data seeded at build/runtime; offline-capable (static assets only)
**Scale/Scope**: 2 roles, ~15 screens/views, 10 mock patients, 30-day vitals history per patient
